# TDD 重構階段：改善程式碼品質

## 目標

在保持測試通過（綠燈 🟢）的前提下，改善程式碼品質。

---

## 核心原則

### 1. 測試保護原則
每次重構後立即執行測試，確保全部通過。若失敗則立即回滾。

### 2. 小步前進原則
一次只做一個小重構，避免一次改動過多。

### 3. 不強行重構原則
只在真正有改善空間時才重構。程式碼已清晰簡潔時保持原樣，遵循 YAGNI 原則。

### 4. 設計原則遵守
重構時必須遵守 SOLID 設計原則，達到好讀、好維護、好擴充。

### 5. 架構規範遵守
**嚴格遵守** `prompts/rules/程式架構規範.md`，確保分層清晰、職責明確。

### 6. 測試 Warnings 清理
**盡可能清除所有測試 warnings**，保持測試輸出乾淨。warnings 通常指出潛在問題或過時用法。

---

## 重構改善方向

### 1. SOLID 設計原則

#### S - Single Responsibility Principle（單一職責原則）
每個類別/函式只負責一件事。

**範例：**
```python
# ❌ Service 做太多事
class AssignmentService:
    def submit_assignment(self, user_id, content):
        # 驗證權限
        if not self._check_permission(user_id):
            raise PermissionError()
        # 驗證內容
        if len(content) < 10:
            raise ValueError()
        # 儲存資料
        self.repository.save(...)
        # 發送通知
        self._send_email(user_id)

# ✅ 職責分離
class AssignmentService:
    def __init__(self, assignment_repo, permission_validator, notification_service):
        self.assignment_repo = assignment_repo
        self.permission_validator = permission_validator
        self.notification_service = notification_service
    
    def submit_assignment(self, user_id, content):
        self.permission_validator.validate(user_id)  # 權限驗證交給專門的類別
        assignment = Assignment(user_id=user_id, content=content)
        self.assignment_repo.save(assignment)
        self.notification_service.notify(user_id)  # 通知交給專門的服務
```

#### D - Dependency Inversion Principle（依賴反轉原則）
高層模組不應依賴低層模組，兩者都應依賴抽象。

**範例：**
```python
# ✅ Service 透過建構子注入 Repository
class LessonProgressService:
    def __init__(self, lesson_progress_repo, journey_subscription_repo):
        self.lesson_progress_repo = lesson_progress_repo
        self.journey_subscription_repo = journey_subscription_repo
    
    def update_progress(self, user_id, lesson_id, progress):
        # 業務邏輯使用注入的 repository
        subscription = self.journey_subscription_repo.find_by_user(user_id)
        if not subscription:
            raise SubscriptionNotFoundError()
        # ...
```

### 2. 架構規範遵守

#### 分層架構檢查

**Controller (app/api/)**：
- ✅ 只處理 HTTP Request/Response
- ✅ 呼叫 Service 執行業務邏輯
- ❌ 不包含業務邏輯
- ❌ 不直接操作 Repository

**Service (app/services/)**：
- ✅ 包含業務邏輯和規則
- ✅ 透過建構子注入 Repository
- ❌ 不直接操作資料庫
- ❌ 不處理 HTTP 細節

**Repository (app/repositories/)**：
- ✅ 封裝資料存取邏輯
- ✅ 使用 SQLAlchemy Session
- ❌ 不包含業務邏輯

#### 範例：符合架構規範的重構

**重構前（違反分層架構）：**
```python
# app/api/assignment.py
@router.post("/api/assignments")
def submit_assignment(request, db: Session = Depends(get_db)):
    # ❌ Controller 包含業務邏輯
    subscription = db.query(JourneySubscription).filter_by(
        user_id=request.user_id
    ).first()
    if not subscription:
        raise HTTPException(404, "未訂閱課程")
    
    # ❌ Controller 直接操作資料庫
    assignment = Assignment(user_id=request.user_id, content=request.content)
    db.add(assignment)
    db.commit()
    return {"status": "success"}
```

**重構後（符合分層架構）：**
```python
# app/api/assignment.py
@router.post("/api/assignments")
def submit_assignment(request, current_user_id: int = Depends(get_current_user_id)):
    # ✅ Controller 只處理 HTTP，呼叫 Service
    service = get_assignment_service()  # 透過依賴注入取得
    assignment = service.submit_assignment(current_user_id, request.content)
    return {"assignment_id": assignment.id, "status": "success"}

# app/services/assignment_service.py
class AssignmentService:
    def __init__(self, assignment_repo, journey_subscription_repo):
        self.assignment_repo = assignment_repo
        self.journey_subscription_repo = journey_subscription_repo
    
    def submit_assignment(self, user_id, content):
        # ✅ Service 包含業務邏輯
        subscription = self.journey_subscription_repo.find_by_user(user_id)
        if not subscription:
            raise SubscriptionNotFoundError()
        
        # ✅ 透過 Repository 操作資料
        assignment = Assignment(user_id=user_id, content=content)
        return self.assignment_repo.save(assignment)
```

### 3. 程式碼品質改善

#### Early Return 原則
減少巢狀，提升可讀性。

```python
# ❌ 深層巢狀
def process(data):
    if data:
        if data.is_valid():
            if data.has_permission():
                return process_data(data)
            else:
                raise PermissionError()
        else:
            raise ValueError()
    else:
        raise DataError()

# ✅ Early return
def process(data):
    if not data:
        raise DataError()
    if not data.is_valid():
        raise ValueError()
    if not data.has_permission():
        raise PermissionError()
    
    return process_data(data)
```

#### 靜態屬性優化
重複使用的資料提升為類別屬性，減少記憶體消耗。

```python
# ❌ 每次調用都創建
class Service:
    def process(self, status):
        mapping = {"A": "狀態A", "B": "狀態B"}  # 每次都創建
        return mapping.get(status)

# ✅ 類別靜態屬性
class Service:
    STATUS_MAPPING = {"A": "狀態A", "B": "狀態B"}  # 只創建一次
    
    def process(self, status):
        return self.STATUS_MAPPING.get(status)
```

#### 明確的返回類型
使用 TypedDict 定義返回資料結構。

```python
# ❌ 模糊的 dict
def get_data() -> dict:
    return {"name": "Alice", "age": 30}

# ✅ 明確的 TypedDict
from typing import TypedDict

class UserData(TypedDict):
    name: str
    age: int

def get_data() -> UserData:
    return {"name": "Alice", "age": 30}
```

#### 命名清晰化
```python
# ❌ 不清楚的命名
def process(data):
    result = data.get("id")
    return result

# ✅ 清晰的命名
def extract_user_id_from_context(context_data):
    user_id = context_data.get("user_id")
    return user_id
```

#### 消除重複邏輯
```python
# ❌ 重複的驗證邏輯
def update_progress(...):
    subscription = self.repo.find_by_user(user_id)
    if not subscription:
        raise SubscriptionNotFoundError()
    # ...

def submit_assignment(...):
    subscription = self.repo.find_by_user(user_id)
    if not subscription:
        raise SubscriptionNotFoundError()
    # ...

# ✅ 提取共用方法
def _validate_subscription(self, user_id):
    subscription = self.journey_subscription_repo.find_by_user(user_id)
    if not subscription:
        raise SubscriptionNotFoundError()
    return subscription

def update_progress(self, user_id, ...):
    subscription = self._validate_subscription(user_id)
    # ...

def submit_assignment(self, user_id, ...):
    subscription = self._validate_subscription(user_id)
    # ...
```

## E2E 重構項目

### 1. Meta 註記清理

#### 刪除的內容
- `# [事件風暴部位: ...]`
- `# [生成參考 Prompt: ...]`

#### 保留的內容
- Given/When/Then/And 業務邏輯描述
- 必要的技術註解（如 "Extract variables for readability"）

#### 範例

**重構前：**
```python
def test_成功提交作業(self, api_client, context, jwt_helper):
    # Given 學員已訂閱課程
    # [事件風暴部位: Aggregate - JourneySubscription]
    # [生成參考 Prompt: Aggregate-Given-Handler.md]
    
    student_id = context["student_id"]
    
    # When 學員提交挑戰題作業
    # [事件風暴部位: Command - submit_assignment]
    # [生成參考 Prompt: Command-Handler.md]
    
    token = jwt_helper.generate_token(student_id)
    response = api_client.post(
        "/api/assignments",
        headers={"Authorization": f"Bearer {token}"},
        json={"content": "我的作業內容"}
    )
    
    # Then 操作成功
    # [生成參考 Prompt: Success-Failure-Handler.md]
    assert response.status_code == 201
```

**重構後：**
```python
def test_成功提交作業(self, api_client, context, jwt_helper):
    # Given 學員已訂閱課程
    student_id = context["student_id"]
    
    # When 學員提交挑戰題作業
    token = jwt_helper.generate_token(student_id)
    response = api_client.post(
        "/api/assignments",
        headers={"Authorization": f"Bearer {token}"},
        json={"content": "我的作業內容"}
    )
    
    # Then 操作成功
    assert response.status_code == 201
```

### 2. Conftest Fixtures 架構

#### 原則
E2E 測試依賴 `conftest.py` 提供的 fixtures，不需要額外提取。

#### 標準 Fixtures
```python
# conftest.py 應提供的核心 fixtures
- db_session: 資料庫 session
- api_client: HTTP client
- context: 測試上下文字典
- jwt_helper: JWT token 生成器
- *_repository: 各種 Repository（已注入 db_session）
```

#### Feature Background 範例
```python
@pytest.fixture(scope="function", autouse=True)
def feature_background(db_session, context, user_repository, journey_repository):
    # Given 系統中有學員 "Alice"
    user = User(id=1, name="Alice", email="alice@example.com")
    user_repository.save(user)
    
    # And 系統中有課程 "Python 入門"
    journey = Journey(id=1, title="Python 入門")
    journey_repository.save(journey)
    
    # 將 IDs 存入 context
    context["student_id"] = 1
    context["journey_id"] = 1
```

## 重構原則總結

✅ **應該做：**
- 清理 Meta 註記
- 保留業務邏輯註解
- 依賴 conftest.py fixtures
- 遵守 SOLID 設計原則
- 嚴格遵守架構規範（分層清晰、依賴注入）

❌ **不應該做：**
- 過度提取（conftest.py 已足夠）

- 強行簡化可讀性
- 違反架構規範（如 Controller 包含業務邏輯）
- 忽略測試 warnings
